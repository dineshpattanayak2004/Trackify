2.3 Authentication Middleware (src/middleware/auth.js)

Purpose: Protect routes and verify user identity

Two Main Functions:
  1. verifyToken: Checks JWT token from request header
    • Extracts Bearer token from Authorization header
    • Verifies token with JWT_SECRET
    • Supports both User and Distributor roles
    • Attaches user info to req.user
  2. requireRole: Checks if user has required role (admin, user, distributor)

🔐 6. Complete Authentication Flow

6.1 Registration Flow

User fills form → Frontend sends POST /auth/register
↓ Backend checks if email exists (Prisma query)
↓ If not exists → Create user in database
↓ Return success response

6.2 Login Flow

User enters credentials → POST /auth/login
↓ Backend finds user by email (Prisma)
↓ Compare password
↓ If valid → Create JWT token
↓ Return token + user info
↓ Frontend stores token in localStorage
↓ Set axios default header
↓ Redirect to /dashboard

6.3 Protected Route Access

User clicks /dashboard
↓ Frontend checks if token exists in localStorage
↓ If yes → Allow access to page
↓ If no → Redirect to /login
↓ API calls include token in header
↓ Backend middleware (verifyToken) verifies token
↓ If valid → Process request
↓ If invalid → Return 401 error

💡 11. Key Concepts Explained

What is JWT (JSON Web Token)?

JWT is a secure way to authenticate users. When a user logs in, the server creates a token that contains user information. This token is sent to the frontend and stored in localStorage. For every subsequent API request, the token is sent in the Authorization header. The server verifies the token and processes the request.

What is Prisma?

Prisma is an ORM (Object-Relational Mapping) tool that makes database operations easier. Instead of writing raw SQL queries, you use JavaScript/TypeScript code to interact with the database. Prisma handles all the complex database operations for you.

What is SQLite?

SQLite is a file-based database. Unlike MySQL or PostgreSQL, it doesn't require a separate database server. The entire database is stored in a single file (dev.db). It's perfect for small to medium applications and development.

What is Socket.io?

Socket.io enables real-time, bidirectional communication between frontend and backend. It's used in Trackify to send live analytics updates every 3 seconds without the frontend needing to refresh or make repeated API calls.

What is CORS?

CORS (Cross-Origin Resource Sharing) allows a server to indicate any origins (domain, scheme, or port) other than its own from which a browser should permit loading of resources. In Trackify, the backend enables CORS so the frontend (port 5173) can make requests to the backend (port 4000).
